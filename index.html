How it works

Files are read locally using the FileReader API — nothing leaves your machine. The first 32–512 bytes are extracted and compared against a prioritized signature table. Signatures with offsets (e.g. ZIP-based formats) are checked at their correct byte position.

Extending the database

All signatures live in signatures.js as a plain JavaScript array. Each entry follows a simple schema: { name, ext, category, magic, offset, desc, mime }. Add your own entries and the analyzer picks them up immediately.

Limitations

Magic bytes identify format, not content. A JPEG is a JPEG regardless of what the image shows. Some formats share headers (e.g. DOCX and EPUB are both ZIP). HexScan resolves these with secondary signature checks at known offsets.

Deploy to GitHub Pages

Push the three files (index.html, style.css, signatures.js) to a repository and enable GitHub Pages from Settings → Pages → Deploy from branch. No build step required.
